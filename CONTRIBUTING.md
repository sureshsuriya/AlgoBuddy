# ↪️ Getting Started

Follow these steps to set up the project locally.

## 1. Fork the Repository

Click the **Fork** button at the top-right corner of this repository.

## 2. Clone Your Fork

```bash
git clone https://github.com/your-username/AlgoBuddy.git
```

## 3. Navigate to the Project Directory

```bash
cd AlgoBuddy
```

## 4. Install Dependencies

```bash
npm install
```

## 5. Start the Development Server

```bash
npm run dev
```

The application will start locally at:

```bash
http://localhost:3000
```

# ↪️ Development Workflow

Follow the workflow below while contributing to the project.

## 1. Create a New Branch

Create a separate branch before making any changes.

### Syntax

```bash
git checkout -b feature/your-feature-name
```

### Example

```bash
git checkout -b fix/navbar-responsive-issue
```

---

## 2. Make Your Changes

You can now start working on:

* Bug fixes
* UI/UX improvements
* Documentation updates
* Performance enhancements
* New visualizers

---

## 3. Commit Your Changes

Write clear and meaningful commit messages.

### Syntax

```bash
git commit -m "type: short-description"
```

### Example

```bash
git commit -m "fix: improve navbar responsiveness"
```

### Recommended Commit Types

| Type       | Purpose                  |
| ---------- | ------------------------ |
| `feat`     | New feature              |
| `fix`      | Bug fix                  |
| `docs`     | Documentation updates    |
| `style`    | UI/styling changes       |
| `refactor` | Code improvements        |
| `perf`     | Performance optimization |

---

## 4. Push Your Changes

Push your branch to your forked repository.

```bash
git push origin feature/your-branch-name
```

---

## 5. Open a Pull Request

After pushing your changes:

1. Open your fork on GitHub
2. Click **Compare & Pull Request**
3. Add a clear title and description
4. Submit the pull request

## Issue Assignment Process

Before starting work on any feature or bug fix, please follow these steps to get properly assigned:

### 1. Find or Create an Issue
- Browse the [open issues](https://github.com/PankajSingh34/AlgoBuddy/issues) to find something you'd like to work on.
- If the bug or feature doesn't have an issue yet, create one first and wait for maintainer feedback before proceeding.

### 2. Request Assignment
- Comment on the issue with something like: `"I'd like to work on this. Can I be assigned?"`
- **Do not submit a pull request without being assigned** — unassigned PRs may be closed.
- Wait for a maintainer to officially assign the issue to you before starting work.

### 3. One Issue at a Time
- Please work on **one issue at a time**. Avoid requesting multiple assignments simultaneously to keep things fair for all contributors.

### 4. Inactivity Policy
- If you are assigned but have not made progress or provided an update within **7 days**, the issue may be unassigned and reopened for others.
- If you need more time, comment on the issue to let maintainers know.

### 5. Good First Issues
- Issues labeled [`good first issue`](https://github.com/PankajSingh34/AlgoBuddy/labels/good%20first%20issue) are specifically reserved for first-time contributors.
- If you're new here, these are the best place to start!

### 6. Need Help?
- If you're stuck or have questions, comment directly on the issue or start a [Discussion](https://github.com/PankajSingh34/AlgoBuddy/discussions).
- Don't hesitate to ask — the community is here to help.

# ↪️ Pull Request Guidelines

Before submitting a PR:

* Ensure the project runs correctly
* Test changes on multiple screen sizes
* Follow clean coding practices
* Avoid unnecessary files
* Use meaningful commit messages
* Keep PRs focused on a single issue/topic
* Add screenshots for UI-related changes whenever possible
